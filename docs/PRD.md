# PRD - 食譜收藏夾

## 1. 專案概述
- **背景與動機**：現代人經常透過網路尋找料理靈感，但散落各處（如部落格、社群媒體、影片等）的食譜難以集中管理與尋找。因此，我們需要一個「食譜收藏夾」來幫助使用者有系統地整理自己的美食資料庫。
- **目標用戶**：
  - **一般人（一般使用者）**：想要有條理地保存各式食譜、有分享交流需求的自煮愛好者。
  - **管理員**：負責維護網站內容品質與系統穩定營運的管理者。
- **核心價值主張**：提供一個直覺、易用且能集中儲存與分享的食譜管理平台，讓記錄料理靈感變得更簡單。

---

## 2. 功能需求

以下列出系統的 5 項核心主要功能與其對應的使用者故事（User Story）：

1. **儲存食譜**
   - 作為 **一般人**，我希望 **能夠將我喜歡的食譜詳細內容（含食材、步驟）儲存到個人的收藏夾中**，以便 **我日後可以隨時查看並跟著作菜**。
2. **收集資料（匯入/自建）**
   - 作為 **一般人**，我希望 **能夠透過簡單的介面記錄或收集網路上的食譜資料重點**，以便 **我能將散落各地的料理資訊統整在同一個平台**。
3. **分享於大眾**
   - 作為 **一般人**，我希望 **能夠將我的私人食譜設定為「公開」**，以便 **其他大眾使用者也能看到我的分享，促進社群交流**。
4. **搜尋與分類探索**
   - 作為 **一般人**，我希望 **能夠透過關鍵字搜尋或是瀏覽不同的分類標籤（如：中式、甜點、快速料理）**，以便 **我在不知道要做什麼菜時能快速找到靈感與靈感食譜**。
5. **平台內容管理**
   - 作為 **管理員**，我希望 **能夠查看、隱藏或刪除平台上不適當的公開食譜與處理用戶管理**，以便 **維持網站的優良內容品質與良好的社群環境**。

---

## 3. 非功能需求

- **技術限制**：
  - 後端框架：使用 Python **Flask**。
  - 前端渲染：使用 **Jinja2** 模板引擎搭配基本的 HTML/CSS/JavaScript。
  - 資料庫：使用 **SQLite** 進行資料儲存，方便輕量化部署。
- **效能與安全考量**：
  - 基本防護：表單輸入需實作防禦措施，防止如 SQL 注入（SQL Injection）與跨站腳本攻擊（XSS）。
  - 效能要求：公開頁面（如食譜牆）在一般資料量下，頁面載入時間需保持在合理的使用者體驗範圍（如 2 秒以內）。
  - 使用者狀態：需利用 Flask Session 妥善管理使用者登入狀態。

---

## 4. MVP 範圍 (最小可行性產品)

- **Must Have (必須有)**：
  - 基礎會員系統（註冊、登入、登出）
  - 新增食譜（輸入標題、圖片連結、食材清單、製作步驟）
  - 我的收藏夾（查看自己儲存的食譜列表與詳細內容）
- **Should Have (應該有)**：
  - 食譜公開/私密設定
  - 公開食譜分享牆（讓不特定大眾也可以瀏覽所有被設為公開的食譜）
  - 基本的關鍵字搜尋功能
- **Nice to Have (加分項)**：
  - 分類標籤（Tags）功能
  - 從已知網址自動抓取/收集外部食譜資料的爬蟲小工具
  - 收藏或按讚他人的公開食譜
